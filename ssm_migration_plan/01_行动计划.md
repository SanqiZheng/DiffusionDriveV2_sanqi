# 行动计划

## 主定义

行动计划的目标，是在不改变 J6 SSM 输入链路的前提下，用新增编码器把现有 SSM 输入编码成 DiffusionDriveV2 `TrajectoryHead` 需要的条件张量，再逐步完成 PyTorch 对齐、ONNX 导出、HBM 编译和端侧接入。

## 最终记忆版本

`只移植 DiffusionDriveV2 规划头；先训练 SSMConditionEncoder 对齐 TrajectoryHead 输入，再做 ONNX/HBM，最后接入 SSM fallback 链路。`

## 当前共识约束

```text
不改变原始 DiffusionDriveV2 网络语义。
不新增 LocalRoad/Decision 语义预测头。
不要求 DiffusionDriveV2 直接输出 centerline / boundary / stop fence / nudge boundary。
只把 DiffusionDriveV2 的原始轨迹类输出作为 adapter 的后处理输入。
```

允许的最小工程改动：

```text
允许写 deployment wrapper。
不接入 DiffusionDriveV2 原始 camera/lidar 感知 backbone。
不要求 SSM 模块输入三前向相机图像、点云或 NAVSIM BEV raster。
不改 TrajectoryHead 的数学语义。
不新增输出头。
只把 forward_test_rl 内部已存在的 traj_to_score 暴露为 candidate_trajectories，并把 traj_to_score[:,-1] 暴露为 selected_trajectory。
```

## 阶段 0：冻结边界

结论：

```text
第一阶段在 SSM module 内新增 DiffusionDriveV2 planning-head shadow 分支。
第一阶段不替换 topic/input slot/output slot。
第一阶段不直接替换 SDPlanner。
第一阶段不强行沿用旧 det/map/ego adapter。
第一阶段不改原有规则链路。
```

需要固定的工程边界：

```text
现有分支:
FW_SSM_LOCAL_ROAD
  -> 原有规则黑盒
  -> LocalRoad + Decision
  -> OUTPUT_SLOT_SSM_LOCAL_ROAD
  -> GridMapModule
  -> OUTPUT_SLOT_SSM_ROAD_PLUS
  -> SDPlanningModule

新增 shadow 分支:
SSM 原始输入
  -> SSMConditionEncoder
  -> DiffusionDriveV2 TrajectoryHead
  -> candidate_trajectories / candidate_scores / selected_trajectory
```

## 阶段 1：离线输入样本构建

目标：从现有 SSM 输入中导出一批可重复回放的模型输入样本。

必需输入：

| 输入类别 | 来源 | 作用 |
|---|---|---|
| 自车状态 | `VSTATUS_DATA` | 构造 ego condition |
| 里程计 | `ODOMETRY` | 坐标对齐、ego motion |
| 泊车定位 | `NDM_SP_LOCATION` | submap、全局到局部转换 |
| 泊车参考线 | `NDM_SP_NAVIGATION` | route/goal condition |
| 动态障碍物 | `LIDAR_TRAJ` 或 `ADAS_OBS` | agents condition |
| 静态语义 | `ADAS_STATIC_OBJECT` / `NDM_FUSION` | map/static condition |
| freespace | `APA_FREESPACE_GRIDX` | dense BEV condition |

产物：

```text
ssm_frame_input.npz
  ego_raw
  odom_raw
  parking_loc_raw
  parking_refline_raw
  dynamic_objects_raw
  static_objects_raw
  freespace_grid_raw
  timestamp
```

## 阶段 2：SSMConditionEncoder

目标：把 SSM 原始输入转换为 DiffusionDriveV2 `TrajectoryHead.forward()` 可消费的条件张量。

建议输出：

```text
ego_query:         float32 [B,1,256]
agents_query:      float32 [B,30,256]
bev_feature:       float32 [B,256,H,W]
bev_spatial_shape: int64   [2]
status_encoding:   float32 [B,1,256]
status_feature:    float32 [B,8]
```

最小可行策略：

1. 动态障碍物编码成 `agents_query`。
2. freespace + 参考线 + 静态障碍物 rasterize 成 `bev_feature`。
3. 自车状态和泊车目标编码成 `ego_query/status_encoding`。
4. `status_feature` 保留为 8 维工程状态向量，用于兼容原 head 内部接口。
5. 暂不接入上一帧轨迹和旧 `temp_instance_feature`。

## 阶段 3：DiffusionDriveV2 deployment forward

目标：绕开原始 `camera_feature / lidar_feature / TransfuserBackbone`，直接使用阶段 2 的条件输入调用 `TrajectoryHead`。

新增 wrapper：

```text
DiffusionDriveV2DeploymentCore
  input:
    ego_query
    agents_query
    bev_feature
    bev_spatial_shape
    status_encoding
    status_feature
  output:
    candidate_trajectories
    selected_trajectory
    candidate_scores
    valid_flag
```

必须删除或关闭的研究侧逻辑：

| 逻辑 | 处理 |
|---|---|
| 在线 `PDMScorer` | 端侧关闭，只用于离线训练/验证 |
| `ProcessPoolExecutor` | 端侧关闭 |
| `metric_cache` 依赖 | 端侧关闭 |
| 默认只返回 `reward_dict` | wrapper 暴露内部已有 `traj_to_score` 和 `traj_to_score[:,-1]` |
| 随机增强 `add_mul_noise` | 第一版固定随机种子或关闭 |

## 阶段 4：Shadow 记录与确定性校验

目标：不要让 DiffusionDriveV2 预测 LocalRoad/Decision 语义头，也不在第一阶段修改原有规则输出；只记录模型轨迹并做确定性校验。

DiffusionDriveV2 提供：

```text
selected_trajectory: [B,8,3]
candidate_trajectories: [B,G,8,3]
candidate_scores: [B,G]
```

shadow 阶段只负责：

```text
记录输出
检查 shape / NaN / 坐标范围 / 抖动
与 baseline 或回放标签对齐评估
```

明确禁止：

```text
DiffusionDriveV2 不直接输出 centerline_points。
DiffusionDriveV2 不直接输出 left/right_boundary_points。
DiffusionDriveV2 不直接输出 stop fence。
DiffusionDriveV2 不直接输出 nudge boundary。
```

## 阶段 5：SSM Adapter 接入

目标：在 `FW_SSM_LOCAL_ROAD` 内接入新 shadow 分支，原有规则黑盒继续输出 baseline。

推荐流程：

```text
baseline = 原有规则黑盒输出
e2e_result = DiffusionDriveV2 shadow 分支输出
publish baseline
record e2e_result
```

稳定后再进入 switch：

```text
if e2e_result valid:
  allow model trajectory related output
else:
  baseline / fallback
```

## 阶段 6：ONNX / HBM

不要第一步就冲完整 HBM。

顺序：

1. PyTorch 离线样本闭环。
2. `SSMConditionEncoder` PyTorch 与 C++ 对齐。
3. `DiffusionDriveV2DeploymentCore` 导出 ONNX。
4. ONNXRuntime 对齐 PyTorch。
5. 清理不支持算子。
6. PTQ 数据集生成。
7. HBM 编译。
8. 端侧回放验证。

## 第一轮验收标准

最小验收：

1. 固定一帧 SSM 输入，adapter 能构造条件张量。
2. PyTorch wrapper 输出无 NaN/Inf。
3. `candidate_trajectories / candidate_scores / selected_trajectory` shape 正确、无 NaN/Inf。
4. 模型轨迹坐标系与 SSM 局部坐标一致。
5. `valid_flag=false` 时明确给出 `fallback_reason`。
6. shadow 分支不改变原有 `OUTPUT_SLOT_SSM_LOCAL_ROAD -> GridMapModule -> OUTPUT_SLOT_SSM_ROAD_PLUS -> SDPlanningModule` 链路。
