## 2. 模块分层视角

如果从“系统结构”看，这个项目可以分成 6 层。

### 第 1 层：原始数据与场景读取

核心文件：

- `navsim/common/dataloader.py`
- `navsim/common/dataclasses.py`

作用：

- 从 NAVSIM / OpenScene 的原始日志中切出场景片段
- 组装 `Scene` 和 `AgentInput`

### 第 2 层：特征与目标构造

核心文件：

- `navsim/agents/diffusiondrivev2/transfuser_features.py`

作用：

- 把相机、LiDAR、ego 状态变成模型输入
- 把未来轨迹、agent box、BEV map 变成监督目标

### 第 3 层：训练数据缓存与 DataLoader

核心文件：

- `navsim/planning/training/dataset.py`
- `navsim/planning/script/run_dataset_caching.py`

作用：

- 将 feature/target 持久化到 gzip pickle
- 支持训练时直接走 cache，提高吞吐

### 第 4 层：模型主体

核心文件：

- `navsim/agents/diffusiondrivev2/transfuser_backbone.py`
- `navsim/agents/diffusiondrivev2/diffusiondrivev2_model_rl.py`
- `navsim/agents/diffusiondrivev2/diffusiondrivev2_model_sel.py`
- `navsim/agents/diffusiondrivev2/modules/blocks.py`

作用：

- 感知编码
- 扩散轨迹生成
- 轨迹选择打分

### 第 5 层：训练封装

核心文件：

- `navsim/planning/script/run_training.py`
- `navsim/planning/training/agent_lightning_module.py`
- `navsim/agents/diffusiondrivev2/diffusiondrivev2_rl_agent.py`
- `navsim/agents/diffusiondrivev2/diffusiondrivev2_sel_agent.py`

作用：

- Hydra 实例化 agent
- Lightning 组织训练 / 验证循环
- 冻结与解冻不同参数子集

### 第 6 层：闭环评测与奖励

核心文件：

- `navsim/evaluate/pdm_score.py`
- `navsim/planning/simulation/planner/pdm_planner/scoring/pdm_scorer.py`
- `navsim/planning/script/run_pdm_score.py`
- `navsim/planning/script/run_pdm_score_fast.py`

作用：

- 把预测轨迹转成闭环规划质量指标
- 训练时提供 reward
- 评测时给出 PDMS / EPDMS

## 3. 两套核心 agent

### 3.1 Stage I：RL generator agent

文件：

- `navsim/agents/diffusiondrivev2/diffusiondrivev2_rl_agent.py`

职责：

- 载入 DiffusionDrive 预训练权重
- 冻结非 `_trajectory_head` 参数
- 只训练生成轨迹相关模块

### 3.2 Stage II：selector agent

文件：

- `navsim/agents/diffusiondrivev2/diffusiondrivev2_sel_agent.py`

职责：

- 载入 RL 阶段 checkpoint
- 冻结生成器
- 只训练 coarse/fine scorer 相关头和编码器

## 4. 训练链总图

### 4.1 Stage I：RL generator

```text
run_training.py
  -> instantiate(cfg.agent=diffusiondrivev2_rl_agent)
  -> AgentLightningModule
  -> Dataset / CacheOnlyDataset
  -> agent.forward(features, targets, metric_cache, token)
  -> diffusiondrivev2_model_rl.V2TransfuserModel.forward
  -> TrajectoryHead.forward_train_rl 先 rollout 轨迹并算 reward/advantage
  -> TrajectoryHead.get_rlloss 再对同一条扩散链回传 RL + IL loss
  -> optimizer.step
```

### 4.2 Stage II：selector

```text
run_training.py
  -> instantiate(cfg.agent=diffusiondrivev2_sel_agent)
  -> AgentLightningModule
  -> Dataset / CacheOnlyDataset
  -> agent.forward(features, targets, metric_cache, token)
  -> diffusiondrivev2_model_sel.V2TransfuserModel.forward
  -> 先 no_grad 生成多轨迹候选
  -> 再训练 coarse scorer + fine scorer
  -> optimizer.step
```

## 5. 推理链总图

### 5.1 生成器推理链

```text
camera/lidar/status
  -> TransfuserBackbone
  -> key/value memory + query tokens
  -> TrajectoryHead.forward_test_rl
  -> 从 plan_anchor + truncated noise 开始
  -> 2-step DDIM denoising
  -> 得到一批候选轨迹
```

### 5.2 selector 推理链

```text
候选轨迹
  -> coarse scorer
  -> top-k
  -> fine scorer
  -> 选出 coarse best + 每层 fine best
  -> 离线评测时再用 PDM reward 打日志
  -> 官方评测时理论上应返回最终 trajectory
```

## 8. 这个工程和普通“单模型训练项目”最大的不同

最大的不同不是网络结构，而是训练外环：

- reward 不是直接从 tensor 上算出来的简单 differentiable loss
- 而是要经过 `Trajectory -> PDM simulator -> PDM scorer -> reward`

这意味着：

1. 模型训练依赖 metric cache
2. 训练过程中存在大量 CPU / multiprocessing / pickle / lzma
3. 生成模型、闭环评测和数据缓存被强耦合在一起

所以如果你后面自己二次开发，最值得优先理解的不是某个单独层，而是：

- `TrajectoryHead`
- `pdm_score_para`
- `CacheOnlyDataset`
- `run_pdm_score_fast`

这四者之间的连接方式。
