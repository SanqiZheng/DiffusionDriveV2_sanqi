# 00 代码阅读路线

这份文档只做一件事：

- 帮你按最省脑力的顺序阅读 DiffusionDriveV2 源码

如果你直接从大模型文件往下啃，很容易迷路。更稳的方式是先建立一条“训练入口 -> 数据 -> 输入表示 -> 生成器 -> 选择器”的主链。

---

## 1. 最推荐的源码阅读顺序

### 第一阶段：先把训练主链串起来

1. `17_逐文件精读_run_training.md`
2. `18_逐文件精读_dataset.md`
3. `16_逐文件精读_transfuser_features.md`

这一阶段的目标是先回答：

- 训练入口是谁
- batch 从哪里来
- feature/target 长什么样

### 第二阶段：吃透 Stage I 生成器

4. `14_逐文件精读_diffusiondrivev2_model_rl.md`

这一阶段的目标是回答：

- anchor-based truncated diffusion 怎么实现
- reward 怎么从 PDM 变成 advantage
- RL loss 怎么从 diffusion chain 上算出来

### 第三阶段：吃透 Stage II 选择器

5. `15_逐文件精读_diffusiondrivev2_model_sel.md`

这一阶段的目标是回答：

- coarse-to-fine selector 怎么接在生成器后面
- coarse/fine scorer 怎么训练
- 为什么还要引入词表候选和 top-k

## 2. 如果你是第一次看这个项目，建议这样配套阅读

## 3. 五个最值得优先吃透的源码文件

### 3.1 `navsim/planning/script/run_training.py`

先看训练入口，避免后面读模型时不知道它是怎么被调用的。

### 3.2 `navsim/planning/training/dataset.py`

先搞清 batch 结构，否则你很难读懂 RL / selector 为什么能拿到 `metric_cache` 和 `token`。

### 3.3 `navsim/agents/diffusiondrivev2/transfuser_features.py`

先知道输入是什么、target 是什么，再去看模型才不会悬空。

### 3.4 `navsim/agents/diffusiondrivev2/diffusiondrivev2_model_rl.py`

这是论文创新点落地最多的地方。

### 3.5 `navsim/agents/diffusiondrivev2/diffusiondrivev2_model_sel.py`

这是理解 Stage II 选择器和 coarse-to-fine 机制的关键文件。

---

## 4. 你在阅读时最好一直盯住的 6 个问题

1. 当前这个函数的输入张量 shape 是什么
2. 当前这个模块是在做“条件编码、生成、还是打分”
3. 当前这一步是否还在局部坐标系下
4. 这里学习的是 `x_start`、`noise`、还是某个 reward proxy
5. 当前这个 loss 对应论文中的哪一个目标
6. 当前这一步是训练分支、推理分支，还是缓存/数据准备分支

---

## 5. 一条最重要的主线

你真正要建立起来的，不是“记住很多类名”，而是下面这条主线：

```text
Hydra 配置
  -> run_training.py
  -> Dataset / CacheOnlyDataset
  -> feature builder / target builder
  -> AgentLightningModule
  -> RL agent / selector agent
  -> diffusiondrivev2_model_rl.py / diffusiondrivev2_model_sel.py
  -> PDM reward / selector score
```

只要这条线打通，这个项目就不会再显得散。
