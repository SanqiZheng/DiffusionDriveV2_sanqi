# 01 论文主线与核心思想

## 1. 论文想解决什么问题

论文要解决的是：

端到端自动驾驶里的扩散式轨迹生成器，虽然能生成多模态候选轨迹，但经常陷入一个两难：

- 要么像普通 diffusion 那样模式坍塌，只会给出一条保守轨迹
- 要么像 DiffusionDrive 那样依靠 anchor 保住多模态，但会同时生成很多低质量、碰撞或越界的轨迹

论文认为，问题根源不在“扩散模型不会生成多模态”，而在“只靠 imitation learning 时，只有一条 GT 轨迹被监督，其他模式没有被真正约束”。  
于是模型虽然学会了“生成很多意图不同的轨迹”，却没有学会“让所有模式都尽量安全且高质量”。

一句话说：

DiffusionDriveV2 要解决的是“多模态生成保住了，但整体质量不稳”的问题。

## 2. 它的核心方法是什么

论文把方法拆成三个层次：

### 2.1 截断扩散生成器

它沿用了 DiffusionDrive 的核心生成范式：

- 用 `KMeans` 从真实轨迹中聚出 `Nanchor=20` 个 anchor
- 每个 anchor 对应一种驾驶意图子空间
- 不从纯高斯白噪声开始，而是从“anchor 加噪”开始
- 只做很少的 denoising steps，也就是“truncated diffusion”

直觉上，这样做相当于：

- 先把“大方向”交给 anchor 决定
- 再让扩散模型在这个 anchor 附近细修轨迹

所以它本质上还是 diffusion，但不是“从零开始胡乱采样”，而是“在带语义的锚点附近做条件去噪”。

### 2.2 用强化学习替代纯 imitation learning 的弱约束

论文认为，原版 DiffusionDrive 的训练本质上还是 IL：

- 每个场景只有 1 条 GT 轨迹
- 训练时只会把最接近 GT 的那个 anchor/mode 当正样本
- 其他 mode 基本没有强监督

所以 DiffusionDriveV2 把 denoising chain 看成一个 MDP：

- state：当前 noisy trajectory `τ_t`
- action：从 `τ_t` 走到 `τ_(t-1)` 的 denoising update
- policy：扩散模型在当前 step 输出的“去噪动作”
- reward：最终轨迹经过闭环规划指标评估后的分数

这一步很关键。它意味着训练目标不再只是“像 GT”，而是“最终规划质量高”。

### 2.3 专门为 anchor-truncated diffusion 设计的 GRPO 变体

论文的真正创新点不是“用了 RL”这么简单，而是：

普通 GRPO 直接拿来用会出问题，因为不同 anchor 代表不同驾驶意图。

例如：

- “直行”
- “左转”
- “变道”

这些不是同一个语义簇里的轻微变化，而是本来就应该共存的不同模式。  
如果把这些 mode 放在同一组里做相对优势比较，模型会倾向把少见但合理的模式压掉，重新走向 mode collapse。

所以论文提出两层设计：

1. `Intra-Anchor GRPO`
   - 只在同一个 anchor 内部比较样本优劣
   - 避免不同驾驶意图之间互相“内卷”

2. `Inter-Anchor Truncated GRPO`
   - 仍然要让全局学习信号有跨 mode 的安全约束
   - 但不直接做跨 mode 的完整相对排序
   - 而是只保留正向改进，并对碰撞等绝对失败行为施加强惩罚

这就是论文标题里 `Reinforcement Learning-Constrained Truncated Diffusion` 的核心含义。

## 3. 相比已有方法，关键创新点是什么

相对普通 diffusion planner：

1. 不是从单一高斯噪声生成，而是从 anchor-based truncated diffusion 开始
2. 明确把多模态规划问题建成“不同驾驶意图子空间”
3. 不再只优化 imitation objective，而是直接优化闭环规划质量

相对原版 DiffusionDrive：

1. 用 RL 给负模式加约束，不再只监督一个正模式
2. 用 `scale-adaptive multiplicative noise` 做探索，而不是逐点加性噪声
3. 用 `Intra-Anchor GRPO` 防止跨意图比较导致 mode collapse
4. 用 `Inter-Anchor Truncated GRPO` 给全局安全性和质量提供额外约束
5. 追加两阶段 `mode selector`，从候选中更稳定地选出高质量轨迹

## 4. 训练目标、推理过程、条件输入、输出对象

### 4.1 训练目标

论文层面有两阶段训练：

#### 阶段 I：训练生成器

目标是同时优化：

- RL loss：让最终轨迹在闭环指标上更优
- IL / BC regularization：防止纯 RL 把基础驾驶能力带偏

直觉上：

- RL 负责“探索更优轨迹”
- IL 负责“别学歪”

#### 阶段 II：训练模式选择器

目标是从多模态候选轨迹里挑出最 goal-aligned 的那条。

它使用：

- BCE loss 学习离散/二值性质的子指标
- Margin-Rank loss 学习连续质量的相对排序

### 4.2 推理过程

推理时的过程是：

1. 从多个 anchor 出发
2. 对每个 anchor 做少步数截断扩散采样
3. 得到一批候选轨迹
4. 用 selector 对候选轨迹打分
5. 选出最优轨迹作为最终输出

### 4.3 条件输入

论文与代码都表明，条件输入主要来自：

- 前向相机图像
- LiDAR BEV 表征
- ego 当前状态与驾驶命令
- 周围目标、地图上下文经过 backbone 编码后的特征

### 4.4 输出对象

生成器直接输出的是：

- 一组未来 ego 轨迹候选

选择器输出的是：

- 对这些候选的质量打分
- 最终选出的轨迹

## 5. 它到底属于哪一类生成模型

如果按“diffusion / flow matching / score-based / VAE / autoregressive / latent model”来分：

它属于：

- `diffusion model`

更准确地说，是：

- `anchor-conditioned truncated diffusion model`
- 工作在“轨迹坐标空间”，不是 latent diffusion
- 在实现里学习的是 `x_start`，也就是去噪后的轨迹样本本身，而不是噪声 `epsilon`

这点很重要，因为后面看代码时你会看到：

- `DDIMScheduler(... prediction_type="sample")`

这就意味着模型预测的是“样本本身”，不是噪声。

## 6. 面向初学者的论文理论主线摘要

你可以把整篇论文先理解成下面这条主线：

### 第一步：先承认自动驾驶天然是多解问题

很多场景本来就不该只有一个答案。

例如路口：

- 可以直行
- 可以等待再变道
- 可以保守跟车
- 可以更积极地绕行

所以模型如果只会输出一条轨迹，往往不是“更聪明”，而是“表达能力不够”。

### 第二步：普通 diffusion 容易塌成一条轨迹

vanilla diffusion 在规划里很容易只学到一个主流模式，其他可能性都被平均掉。  
DiffusionDrive 用 anchor 解决了这个问题：先人为把轨迹空间切成多个意图子空间，再在每个子空间附近采样。

### 第三步：但只靠 imitation learning，很多 mode 没被真正教好

问题来了：

每个训练样本只有 1 条 GT。  
于是训练时总会偏向“离 GT 最近的那个 mode”，其他 mode 缺乏明确约束。

结果就是：

- 多样性有了
- 但质量不稳定
- 低质量轨迹大量存在

### 第四步：要让每个 mode 都对“结果质量”负责

这就是 RL 出场的原因。

论文的想法不是“放弃 imitation learning”，而是：

- 先用 IL 给模型一个合理起点
- 再让 RL 按闭环质量去修正所有模式

这样模型就不再只学“像专家”，而是学“最终规划结果好不好”。

### 第五步：但 RL 也不能胡用，因为不同 anchor 本来就是不同意图

如果把“左转”和“直行”的样本放在一起比优劣，模型会倾向只留下最常见、最安全、最容易拿高分的模式。  
这会重新造成 mode collapse。

所以论文强调：

- 同一 anchor 内部可以比较
- 不同 anchor 之间不能简单直接比较

这就是 `Intra-Anchor GRPO` 的直觉。

### 第六步：同时又需要跨 anchor 的全局安全约束

如果完全不看跨 anchor 的全局信息，又会出现另一个问题：

- 某个 anchor 内部“相对最好”的样本，绝对上可能还是很差

所以论文又加了一层 `Inter-Anchor Truncated GRPO`：

- 不强行要求不同意图之间做完整排序
- 但对明显失败的轨迹直接给惩罚
- 对真正优于基准的轨迹保留正向信号

### 第七步：生成器再强，也最好加一个 selector

因为多模态生成器本来就会给出候选集。  
最终部署时，系统还是要在候选中挑一条最适合当前目标的轨迹。

所以论文最后又加了 coarse-to-fine selector。

## 7. 后面读源码时一定会出现的核心概念索引

下面这些概念在源码里一定会反复出现：

| 论文概念 | 直觉解释 | 你会在代码里看到什么 |
| --- | --- | --- |
| Anchor / GMM prior | 先把轨迹空间切成多个驾驶意图子空间 | `plan_anchor`, `kmeans_navsim_traj_20.npy` |
| Truncated diffusion | 不做长链 denoising，只做少量 step | `DDIMScheduler`, `step_num=2/10`, `trunc_timesteps=8` |
| x_start prediction | 模型直接预测去噪后的轨迹样本 | `prediction_type="sample"` |
| Exploration noise | 训练时故意扰动轨迹，逼模型探索 | `DDIMScheduler_with_logprob.step`, `add_mul_noise` |
| Intra-Anchor GRPO | 只在同 anchor 内做 group-relative advantage | `reward_group.view(bs, num_groups, ego_fut_mode)` |
| Inter-Anchor Truncation | 跨 mode 只保留正收益，失败强惩罚 | `advantages.clamp(min=0)`, collision/drivable-area penalty |
| PDM reward | 用闭环规划指标当 reward | `PDMScorer`, `pdm_score_para`, `metric_cache` |
| Selector | 从候选轨迹里选最优 | `scorer_decoder`, `fine_scorer_decoder`, `topk` |

## 8. 先记住的结论

如果你只先记住一句最重要的话：

DiffusionDriveV2 不是“把 DiffusionDrive 换成 RL”这么简单，而是“保留 anchor-based truncated diffusion 的多模态结构，再用专门适配该结构的 RL 规则去约束所有模式，并最终用 selector 稳定选出最优轨迹”。
