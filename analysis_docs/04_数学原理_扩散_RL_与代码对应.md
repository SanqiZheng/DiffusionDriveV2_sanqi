# 04. 数学原理：扩散、RL 与代码对应

这一篇不追求论文公式的完整复刻，而是追求“你能把公式和代码对上”。

## 1. 这份代码里的数学对象分别是谁

### 1.1 样本

代码对象：

- `x_start`
- `diffusion_output`
- `prev_sample`

数学含义：

- `x_0`: 干净轨迹
- `x_t`: 第 `t` 步带噪轨迹
- `x_{t-1}`: 去噪一步后的轨迹

### 1.2 条件

代码对象：

- `bev_feature`
- `agents_query`
- `ego_query`
- `time_embed`

数学含义：

- 场景条件 `c`
- 时间条件 `t`

所以网络本质上学的是：

`f_theta(x_t, t, c) -> x_0`

## 2. 为什么这里说它是 diffusion，但又不像教程里的 diffusion

因为它同时具备两个特点：

1. 有明确的“加噪 -> 去噪”链
2. 但预测目标不是 `epsilon`，而是 `x_start`

对照代码：

- [diffusiondrivev2_model_rl.py](../navsim/agents/diffusiondrivev2/diffusiondrivev2_model_rl.py) 第 716-728 行附近，scheduler 配置 `prediction_type="sample"`
- 第 527 行开始的 `DDIMScheduler_with_logprob.step`

所以分类上它仍然属于 diffusion / truncated diffusion 系列，而不是 flow matching、autoregressive、VAE。

## 3. anchor 先验的数学直觉

论文和代码都不是直接从纯标准高斯开始，而是从 anchor 分布开始。

如果把第 `k` 个 anchor 写成 `a_k`，那么可以把候选轨迹理解成：

`trajectory = anchor + scene-conditioned offset`

代码对象：

- `self.plan_anchor`
- `poses_reg[..., :2] = noisy_traj_points + residual`

这里的残差学习很重要，因为它让模型优化的是“修正量”，不是整个轨迹绝对坐标。

## 4. 加噪过程在代码里是什么

训练开始时：

1. 先把 anchor 归一化成 `x_0`
2. 采样噪声 `noise`
3. 调 `scheduler.add_noise(original_samples=x_0, noise=noise, timesteps=trunc_timesteps)`

记成公式可以写成：

`x_t = sqrt(alpha_bar_t) * x_0 + sqrt(1 - alpha_bar_t) * noise`

这里 `x_t` 对应代码里的：

- `diffusion_output`

## 5. 去噪过程在代码里是什么

网络输入当前 noisy trajectory `x_t`，输出对 `x_0` 的估计：

`x0_hat = f_theta(x_t, t, c)`

在代码里：

- `x_start = poses_reg[..., :2]`

然后 scheduler 再根据 `x0_hat` 计算：

- `pred_epsilon`
- `prev_sample_mean`
- `prev_sample`

如果是 `prediction_type="sample"`，那么：

`x0_hat = model_output`

`epsilon_hat = (x_t - sqrt(alpha_bar_t) * x0_hat) / sqrt(1 - alpha_bar_t)`

这正是 `DDIMScheduler_with_logprob.step` 里对应的分支。

## 6. 为什么要有乘性噪声

普通加性噪声：

`x' = x + sigma * epsilon`

乘性噪声更像：

`x' = x * (1 + sigma * epsilon)`

项目里实际是“乘性 + 加性”的组合，但加性部分在当前实现里几乎没真正起主要作用，因为：

- `std_dev_t_add` 被设成 `0`

核心仍然是乘性噪声：

`prev_sample = prev_sample_mean * variance_noise_mul`

为什么规划里适合乘性噪声：

1. 远端轨迹本来坐标更大，允许更大探索幅度
2. 近端轨迹更接近当前车辆，改动不宜太猛
3. 乘性噪声天然带有这种“按尺度伸缩”的味道

## 7. reward 的数学结构

PDM score 聚合结构大致可以写成：

`final_score = hard_safety(product) * weighted_quality(sum)`

在代码里就是：

1. `multiplicate_metric_scores = self._multi_metrics.prod(axis=0)`
2. `weighted_metric_scores = (...)`
3. `final_scores = multiplicate_metric_scores * weighted_metric_scores`

这在 [pdm_scorer.py](../navsim/planning/simulation/planner/pdm_planner/scoring/pdm_scorer.py) 第 156 行附近。

## 8. 代码里的 advantage 到底怎么算

这部分一定要和 shape 一起看。

训练里候选总数是：

`num_groups * 20`

reshape 后：

`reward_group.shape = [B, num_groups, 20]`

这里可以把第 3 维的 `20` 理解为 anchor id，把第 2 维的 `num_groups` 理解为“同 anchor 的多次探索样本”。

于是代码计算：

`A = (r - mean_same_anchor) / (std_same_anchor + eps)`

对应实现：

- `mean_grouped_rewards = reward_group.mean(dim=1)`
- `std_grouped_rewards = reward_group.std(dim=1)`

这就是论文里 intra-anchor GRPO 的代码化版本。

## 9. truncated 的数学直觉

作者不希望不同驾驶意图互相卷，比如：

- 左转 anchor
- 直行 anchor

它们本来就不该直接比高低。

所以代码没有做跨 anchor 的完整排序，而是做了截断：

1. 组内做标准化 advantage
2. 只保留优于参考轨迹的样本
3. 硬安全失败则把 advantage 打掉

代码：

- `mask_positive = reward_group > reward_gt`
- `advantages = advantages.clamp(min=0) * mask_positive.float()`
- 对 `no_collision` 和 `drivable_area` 不满足的样本置负

这就是“只给好样本正向信号，同时避免跨意图直接比较”的工程化实现。

## 10. `get_rlloss` 为什么能对应 policy gradient

经典 REINFORCE 的形式是：

`L = - E[ log pi(a|s) * A ]`

这里动作不是离散 action，而是 diffusion 每一步的 transition：

`x_t -> x_{t-1}`

所以代码把每一步 transition 当成策略动作，使用：

`log pi_theta(x_{t-1} | x_t, c)`

然后构造：

`loss ~ - log pi * A`

实际写法是：

`-exp(logp - stopgrad(logp)) * A`

这是因为它需要一个前向值稳定、但梯度等价于 `-logp * A` 的实现技巧。

## 11. 为什么还要保留 IL loss

如果只有 RL：

- reward 稀疏
- 方差大
- 容易把轨迹学歪

所以代码额外加入：

`L_total = L_RL + lambda * L_IL`

其中：

- 有正优势样本时，`lambda = 0.1`
- 没有正优势样本时，`lambda = 1.0`

这代表：

- RL 信号可靠时，监督只做轻微约束
- RL 信号很差时，用监督稳住训练

## 12. yaw 的数学来源

`bezier_xyyaw` 里不是直接学 yaw，而是：

1. 用控制点构造 Bézier 曲线
2. 求一阶导 `B'(t)`
3. 令 `yaw = atan2(dy, dx)`

这相当于：

`theta(t) = atan2( y'(t), x'(t) )`

它的好处是几何一致性强：

- 朝向永远和轨迹切线一致
- 不会出现“轨迹向前，车头却突然歪过去”的不自然现象

## 13. 从数学到代码的一句话映射

可以把整个主目标浓缩成一句话：

“在场景条件 `c` 下，模型学习从 anchor 邻域的带噪轨迹 `x_t` 预测更优的干净轨迹 `x_0`，再把每一步去噪 transition 当成策略动作，用 PDM 闭环分数构造 advantage，通过 policy gradient 方式提升高质量轨迹的生成概率。”
